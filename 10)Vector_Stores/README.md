# 📖 Folder 10: Vector Stores

## 🎯 Objective

Store documents with embeddings in an efficiently queryable format. Enable semantic search through vector similarity.

## 📚 Theory & Concepts

### **Vector Store Architecture**

```
Documents
    ↓ (Chunked by TextSplitters)
Embeddings
    ↓ (Generated by Embedding Models)
Vector Store
    ├── Index (optimized for search)
    ├── Metadata (track sources)
    └── Embeddings (high-dimensional vectors)
    ↓
Fast Similarity Search
    ↓
Retrieved Documents
```

### **How Vector Search Works**

```
Query: "What is machine learning?"
    ↓
Generate Embedding: [0.123, -0.456, 0.789, ...]
    ↓
Compare to all document embeddings (cosine similarity)
    ↓
Top-K most similar results: [Doc1: 0.95, Doc2: 0.88, Doc3: 0.82]
    ↓
Return highest scoring documents
```

### **ChromaDB - Vector Store Used Here**

```
ChromaDB Features:
├── Lightweight (in-memory or persistent)
├── No external server needed
├── Supports metadata filtering
├── Built-in similarity search
├── Collections (organize documents)
└── Perfect for learning & small-scale apps

Persistence:
├── In-memory: Fast, lost on restart
└── Persistent: Saved to disk, survives restart
```

### **Vector Store vs Database**

```
Traditional Database:
├── Exact matching
├── SQL queries
└── Not semantic

Vector Store:
├── Similarity matching
├── Embedding-based search
├── Semantic understanding
└── Approximate search
```

### **Why Embeddings + Storage**

```
Without Vector Store:
Query → Compare to ALL docs → SLOW

With Vector Store:
Query → Index lookup → FAST (logarithmic)

Indexing:
├── Builds on first use
├── Reusable
├── Enables real-time search
```

## 🔑 Key Learning Points

### **1. Initialize Chroma Vector Store**

```python
from langchain_community.vectorstores import Chroma
from langchain_huggingface import HuggingFaceEmbeddings
from langchain_community.document_loaders import TextLoader
from langchain_text_splitters import RecursiveCharacterTextSplitter

# Step 1: Load documents
loader = TextLoader("document.txt")
documents = loader.load()

# Step 2: Split documents
splitter = RecursiveCharacterTextSplitter(
    chunk_size=1000,
    chunk_overlap=200
)
chunks = splitter.split_documents(documents)

# Step 3: Create embeddings
embeddings = HuggingFaceEmbeddings(
    model_name="sentence-transformers/all-MiniLM-L6-v2"
)

# Step 4: Create vector store
vector_store = Chroma.from_documents(
    documents=chunks,
    embedding=embeddings,
    collection_name="my_docs",
    persist_directory="./chroma_db"  # Optional: persistent storage
)

print("✅ Vector store created!")
```

### **2. Similarity Search**

```python
# Query
query = "What is machine learning?"

# Search (returns top 3 similar documents)
results = vector_store.similarity_search(
    query=query,
    k=3
)

# Results
for i, doc in enumerate(results):
    print(f"{i+1}. {doc.page_content[:100]}")
    print(f"   Source: {doc.metadata['source']}\n")
```

### **3. Similarity Search with Scores**

```python
# Get similarity scores along with results
results = vector_store.similarity_search_with_score(
    query="Python programming",
    k=5
)

for doc, score in results:
    print(f"Score: {score:.2f} (0-1, higher=better)")
    print(f"Content: {doc.page_content[:50]}...\n")
```

### **4. Add New Documents**

# Add more documents later
new_documents = [

### **5. Load Existing Store**

```python
from langchain_community.vectorstores import Chroma
from langchain_huggingface import HuggingFaceEmbeddings

embeddings = HuggingFaceEmbeddings()

# Load from persistent storage
vector_store = Chroma(
    collection_name="my_docs",
    embedding_function=embeddings,
    persist_directory="./chroma_db"
)

# Use immediately
results = vector_store.similarity_search("query")
```

### **6. Metadata Filtering**

```python
# Search with metadata filtering
results = vector_store.similarity_search(
    query="What is AI?",
    k=5,
    filter={"source": "ai_guide.txt"}  # Only from specific source
)

# Only returns docs matching both:
# 1. Semantic similarity to query
# 2. source == "ai_guide.txt"
```

## 💡 Practical Patterns

### **Pattern 1: Complete Indexing Pipeline**

```python
from langchain_community.document_loaders import DirectoryLoader, TextLoader
from langchain_text_splitters import RecursiveCharacterTextSplitter
from langchain_community.vectorstores import Chroma
from langchain_huggingface import HuggingFaceEmbeddings

# 1. Load all documents
loader = DirectoryLoader("./docs", glob="**/*.txt", loader_cls=TextLoader)
documents = loader.load()

# 2. Split
splitter = RecursiveCharacterTextSplitter(chunk_size=1000, chunk_overlap=200)
chunks = splitter.split_documents(documents)

# 3. Create embeddings
embeddings = HuggingFaceEmbeddings()

# 4. Store
vectorstore = Chroma.from_documents(
    chunks,
    embeddings,
    persist_directory="./vector_db"
)

print(f"✅ Indexed {len(chunks)} chunks")
```

### **Pattern 2: Search with Context**

```python
# Get similar documents with ranking
query = "How to learn Python?"
docs_with_scores = vector_store.similarity_search_with_score(query, k=5)

print(f"Query: {query}\n")
for rank, (doc, score) in enumerate(docs_with_scores, 1):
    print(f"#{rank} (Score: {score:.3f})")
    print(f"   {doc.page_content[:80]}...")
    print(f"   Source: {doc.metadata['source']}\n")
```

### **Pattern 3: Persistence Management**

```python
import os
from langchain_community.vectorstores import Chroma

persist_dir = "./chroma_db"

# Check if already indexed
if os.path.exists(persist_dir):
    # Load existing
    vectorstore = Chroma(
        collection_name="docs",
        embedding_function=embeddings,
        persist_directory=persist_dir
    )
    print("✅ Loaded existing index")
else:
    # Create new
    vectorstore = Chroma.from_documents(
        chunks,
        embeddings,
        persist_directory=persist_dir
    )
    print("✅ Created new index")
```

### **Pattern 4: Update and Maintain**

```python
# Get existing vector store
vectorstore = Chroma(
    collection_name="docs",
    embedding_function=embeddings,
    persist_directory="./chroma_db"
)

# Add new documents
new_docs = [Document(page_content="...", metadata={...})]
vectorstore.add_documents(new_docs)

# Persist changes
vectorstore.persist()
print("✅ Index updated")
```

## 🎓 Advanced Concepts

### **Index Types in ChromaDB**

```
Flat Index:
├── All vectors stored explicitly
├── Memory: O(n*d) where n=vectors, d=dimensions
├── Speed: Linear search O(n)
└── Used for: Small datasets (<100K vectors)

Hierarchical Index (Future):
├── Tree structure
├── Memory: Still O(n*d)
├── Speed: Logarithmic O(log n)
└── Used for: Large datasets
```

### **Similarity Metrics**

```python
# Default: Cosine similarity (0-1)
# Also supports:
# - L2 distance (Euclidean)
# - Inner product (dot product)

# In Chroma, metric is set during collection creation
```

### **Batch Operations**

```python
# More efficient for large operations
vectorstore.add_documents(large_list_of_docs)  # Batched internally
results = vectorstore.similarity_search(query, k=100)  # Optimized
```

## ⚠️ Common Issues & Solutions

| Issue | Solution |
|-------|----------|
| Store not persisting | Check persist_directory exists |
| Slow search | Reduce k value, pre-filter metadata |
| Memory issues | Use external store (Weaviate, Pinecone) |
| Poor results | Check embedding model, increase chunk_overlap |

## 💡 Pro Tips

1. **Persistence**: Always use `persist_directory` in production
2. **Metadata**: Use metadata filtering for better relevance
3. **Batch Operations**: Add multiple documents at once
4. **k Parameter**: Balance between relevance and speed
5. **Monitoring**: Track number of vectors for scalability planning

## 📈 Scalability Considerations

```
1-10K docs:     ✅ Chroma (in-memory)
10-100K docs:   ✅ Chroma (persistent)
100K-1M docs:   ⚠️ Chroma (slow), better: FAISS
1M+ docs:       ❌ Cloud solution (Pinecone, Weaviate)
```
